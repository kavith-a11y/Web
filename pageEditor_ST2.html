Splash screen

Technical Description

The Splash screen serves as an immersive entry point, utilizing a minimal semantic HTML structure—primarily a central main container—to focus the user's attention. To achieve perfect vertical and horizontal centering of the branding and welcome text, CSS Flexbox (display: flex, justify-content: center, align-items: center) is paired with a full-viewport height unit (100vh). The visual design instantly establishes the aquatic theme using a full-screen background image (background-size: cover, background-position: center). To enhance the introductory user experience (UX) without relying on JavaScript, CSS animations (@keyframes) and transition properties are used to create a smooth fade-in effect (opacity). Finally, a prominently styled anchor link acts as a clear call-to-action, using hover pseudo-classes to provide interactive feedback as it guides the visitor into the main website, while the logo utilizes an alt attribute to maintain strict accessibility standards.

Accessibility

The Splash screen is designed with strict adherence to accessibility standards to ensure an inclusive user experience from the very first interaction. The central logo utilizes a descriptive alt attribute, allowing screen reading software to accurately convey the site's branding to visually impaired users. To maintain strict readability against the full-screen background image, high color contrast is enforced for all welcome text and calls-to-action. The primary entry link is fully keyboard-navigable, enabling users who rely on keyboard controls to easily tab to and activate the button without needing a mouse. Furthermore, the page relies on a clean, semantic document structure using a main landmark, while any CSS transition effects are kept smooth and non-disruptive to accommodate users with motion sensitivities.

Action impact Simulator

Technical Description

The Action Impact Simulator combines CSS styling and JavaScript logic to create a dynamic, interactive experience. The layout organizes clickable action cards using a responsive grid, styled with rgba backgrounds and hover effects to provide immediate visual feedback. The core functionality is driven by a custom JavaScript function triggered by onclick events. This script leverages the classList.toggle method to efficiently track a card's state in a single step, adding points when a card is selected and subtracting them when deselected. Based on the real-time score, conditional if/else statements dynamically update the DOM; the script uses innerHTML to display the current score and impact tier, while simultaneously manipulating the body's backgroundImage property to swap the background visual, providing users with instant, tier-based feedback on their environmental impact.

Accessibility

The Action Impact Simulator ensures accessibility by using descriptive alt attributes on all card images for screen readers. It maintains high text contrast against dynamic backgrounds by using semi-transparent rgba containers. Clear visual feedback through hover effects and a distinct selected state helps users track their choices, while a logical heading structure (h1 to h3) allows assistive technologies to easily navigate the page content.

Content Page

Technical Description

The educational content page uses semantic HTML5 landmarks like header, main, section, and footer for a clear, accessible structure. To help users navigate text-heavy sections, it features internal anchor links and a fixed back-to-top button. Internal CSS is used to style and center educational diagrams with box shadows and rounded borders. Finally, scientific accuracy and accessibility are maintained by using subscript formatting for chemical formulas and descriptive alt attributes for all images.

Accessibility

The educational content page ensures accessibility by using semantic HTML5 landmarks like header, main, and section, allowing screen reading software to easily navigate the document structure. All educational diagrams and logos feature descriptive alt attributes to provide clear context for visually impaired users. Furthermore, internal jump links and a back-to-top button support efficient keyboard navigation, while semantic subscript formatting guarantees that chemical formulas are read accurately by assistive technologies.

Challenges and Lessons Learned

While building the web pages, I ran into a few problems, but fixing them taught me a lot. First, when I put my page into the team leader's template, our CSS styles clashed. I fixed this by using specific class names so my styles wouldn't mess up the rest of the site. Second, the code checker found that my lists were built incorrectly, so I fixed the order of my bullet points and adjusted the spacing. Third, for the splash screen, I struggled to make the background and text look right on both large computer screens and small mobile phones. I solved this by using flexible CSS rules to make the layout adapt to different sizes. Finally, in the Action Impact Simulator, the background color would get stuck if a user's score went back to zero. I fixed this by updating the JavaScript to handle a zero score properly. Overall, these challenges taught me to keep my code organized, make designs that work on all devices, and always test how the site acts when users do unexpected things.
