User Profile

Technical Description

The User Profile page provides an interactive experience by collecting user information through JavaScript prompt() functions, allowing step-by-step data input. A structured semantic HTML layout is used, with sections and containers to organise content clearly. Dynamic content is generated using DOM manipulation methods such as getElementById() and innerHTML, enabling real-time updates of the user’s profile on the page. A completion percentage feature is implemented using basic calculations to enhance user engagement and provide feedback. CSS styling is applied to maintain a clean and consistent layout, while ensuring readability and responsiveness across devices. Accessibility is supported through the use of clear labels, structured headings, and properly defined content sections.

Accessibility

The User Profile page is designed with accessibility in mind to ensure an inclusive and user-friendly experience. Semantic HTML elements such as headings and structured sections are used to improve screen reader navigation and content understanding. All images include descriptive alt attributes to provide meaningful context for visually impaired users. Clear and readable text with sufficient colour contrast is maintained throughout the page to enhance visibility. The interactive profile creation process, including JavaScript prompts, is simple and easy to follow, ensuring usability for a wide range of users. Additionally, all navigation links are keyboard-accessible, allowing users to interact with the page without relying on a mouse. The overall layout is kept clean and consistent to support better readability and accessibility across different devices.

Sitemap

Technical Description

The Sitemap is engineered as a structured navigational hub, utilizing a semantic HTML5 hierarchy to ensure both clarity and SEO optimization. It employs a nested list architecture, styled with CSS Flexbox to maintain a clean, organized layout across various screen dimensions. From a functional standpoint, the sitemap serves as a centralized directory, leveraging anchor tags and internal ID linking to facilitate seamless transitions between the Action Impact Simulator and other core site modules. This technical mapping not only simplifies the user journey but also provides a clear blueprint of the site's information architecture for search engine crawlers.

Accessibility

The Sitemap ensures universal access by adhering to a strict, logical heading structure that begins with a clear h1 and descends to h3, allowing users of assistive technologies like screen readers to easily understand the page hierarchy and jump directly to specific content sections. Every navigational link features high-contrast text and a distinct focus state, ensuring that keyboard-only users can identify their current location within the site map. Furthermore, the use of semantic HTML list elements provides essential context to screen readers, ensuring that the relationship between parent categories and sub-pages is communicated clearly and effectively to all users, regardless of their browsing method.

Content Page

Technical Description

The Sustainable Fishing educational page is engineered using semantic HTML5 landmarks, including header, main, and footer, to establish a clear and organized document structure. To assist users in navigating the text-heavy sections on overfishing and policy, the page features a functional internal navigation menu using anchor tags and a fixed "Go to Top" button for seamless scrolling. Internal CSS is utilized to style educational diagrams with specific dimensions and rounded borders, ensuring a professional visual consistency across the "BlueFuture" platform. Finally, the technical implementation ensures data integrity by using clear list structures for sustainability solutions and responsive layout techniques that adapt to various screen sizes.

Accessibility

The Sustainable Fishing page ensures comprehensive accessibility by utilizing semantic HTML5 landmarks that allow screen reading software to interpret the document’s natural flow. Every educational image regarding marine life and fishing practices includes a descriptive alt attribute, providing essential context for visually impaired users. Furthermore, the inclusion of internal jump links and a prominent back-to-top button facilitates efficient keyboard navigation, while a strict logical heading hierarchy (h1 through h3) allows assistive technologies to skip directly to specific sections like Environmental Impact or Government Policies, ensuring an inclusive learning experience for all.

Challenges and Lessons Learned

The development of the content.html, profile.html, and sitemap.html modules required solving complex integration and logic issues. During the consolidation phase, CSS specificity conflicts occurred between my custom styles and the global template; I resolved this by refactoring my selectors into unique class namespaces to prevent style leakage across the site. Within content.html, the W3C validator identified nesting errors in the navigation lists, which were corrected to ensure a valid Document Object Model tree. For the sitemap.html and splash screen, responsive web design challenges were addressed by implementing fluid CSS layouts and media queries, ensuring visual fidelity across desktop and mobile viewports. Finally, a state-management bug in the Action Impact Simulator where the background failed to reset at a zero score was resolved by updating the JavaScript conditional logic to handle base-case values dynamically.
